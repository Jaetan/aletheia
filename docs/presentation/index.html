What Is Aletheia?


Python library for CAN bus temporal verification


9,700 fps — 1.2× headroom over 1 Mbps CAN bus

O(1) memory — stream gigabyte traces

Proven correct — the checker is mathematically verified


Not "tested correct" — proven correct.
Tests show the presence of bugs. Proofs show their absence.

Architecture


Python — User-facing API, ctypes FFI
What you write. Familiar interface.
↓ shared library (libaletheia-ffi.so)
Haskell — FFI wrapper (68 lines)
No logic. Just data marshalling.
↓ MAlonzo compiled code
Agda — All logic + proofs (36,500 lines)
CAN parsing, signal extraction, LTL checking, DBC validation. Proven correct.

Critical rule: ALL logic in Agda. Haskell = plumbing. Python = interface.

What "Proven Correct" Means


Approach What It Provides What It Misses

Check API

For scripters — industry vocabulary, no LTL knowledge needed



from aletheia import Check

# Simple signal checks
Check.signal("VehicleSpeed").never_exceeds(220)
Check.signal("BatteryVoltage").stays_between(11.5, 14.5)
Check.signal("CoolantTemp").settles_between(80, 100).within(5000)
Check.signal("FaultCode").never_equals(255)
Check.signal("BatteryVoltage").never_below(11.0)

# Causal checks (when/then)
Check.when("BrakePedal").exceeds(50)
     .then("BrakeLight").equals(1).within(100)

Check.when("Ignition").equals(1)
     .then("EngineRPM").exceeds(500).within(2000)

Read it out loud. That's the specification.

Full DSL

For developers — full LTL control, maximum expressiveness



from aletheia import Signal

speed = Signal("VehicleSpeed")
brake = Signal("BrakePressure")

# Safety: speed always below 120
speed.less_than(120).always()

# Liveness: brake response within 500ms
speed.greater_than(80).implies(
    brake.greater_than(0).within(500)
).always()

# Staleness detection (the alive counter check)
Signal("FrameCounter").changed_by(0).never()

# Sensor range
Signal("BatteryVoltage").between(11.0, 14.5).always()

Full Workflow

12 lines: load DBC, define property, stream, detect



from aletheia import AletheiaClient, Signal
from aletheia.dbc_converter import dbc_to_json

dbc = dbc_to_json("vehicle.dbc")
speed_limit = Signal("VehicleSpeed").less_than(120).always()

with AletheiaClient() as client:
    client.parse_dbc(dbc)
    client.set_properties([speed_limit.to_dict()])
    client.start_stream()

    for frame in can_trace:
        response = client.send_frame(
            frame.timestamp_us, frame.can_id, frame.data
        )
        if response.get("status") == "violation":
            print(f"Violation at {response['timestamp']}us")

    client.end_stream()

Multi-Bus Monitoring



from concurrent.futures import ThreadPoolExecutor
from aletheia import AletheiaClient

# One client per CAN bus, independent state
buses = {
    "powertrain": (dbc_pt,  checks_pt,  trace_pt),
    "chassis":    (dbc_ch,  checks_ch,  trace_ch),
    "body":       (dbc_bd,  checks_bd,  trace_bd),
}

def monitor_bus(name, dbc, checks, trace):
    with AletheiaClient(rts_cores=2) as client:
        client.parse_dbc(dbc)
        client.set_properties([c.to_dict() for c in checks])
        client.start_stream()
        for frame in trace:
            response = client.send_frame(...)
        client.end_stream()

with ThreadPoolExecutor(max_workers=len(buses)) as pool:
    futures = {pool.submit(monitor_bus, *args): n
               for n, args in buses.items()}

The Proof Stack

What is formally proven (not tested — proven)


  • CAN encoding roundtrip — encode then decode = original, for all frames
  • Signal extraction — bit-level extraction matches DBC spec, for all signals
  • LTL adequacy — incremental evaluator matches denotational semantics
  • All 13 LTL operators — Always, Eventually, Until, Release, Implies, And, Or, Not, Next, Literal, Atomic, Within, ChangedBy
  • DBC validation — overlapping signals always detected
  • Iteration correctness — streaming = batch (spec equivalence)
  • Parser combinators — all terminate, produce correct output
  • Terminal idempotence — final verdicts are stable (15 lemmas)
