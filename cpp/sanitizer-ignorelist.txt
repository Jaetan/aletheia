# OpenXLSX vendors a fork of miniz called "zippy".  zippy.hpp performs
# unaligned loads on byte buffers (mz_uint32 reads at non-4-byte boundaries),
# which is the well-known miniz pattern; safe on x86_64 where the CPU
# permits it but technically UB per the C++ standard.  Aletheia uses
# OpenXLSX read-only via the optional excel loader; we cannot patch the
# vendored zippy without forking OpenXLSX.  XLStyles.cpp also performs
# bool loads from spreadsheet bytes that contain non-{0,1} values (UBSan
# strictly forbids), and other XL*.cpp files exhibit similar third-party
# UB on adversarial spreadsheet content.  Suppress all OpenXLSX vendored
# code; aletheia-cpp's own source remains fully instrumented.
