### Key Technical Insights

- **Where-block opacity**: Where-block functions create opaque auxiliary definitions. External `with`-matching doesn't cause internal `with` patterns to reduce. Fix: extract to top-level.
- **Metavariable blocking**: `checkRangeBounds-allW` does `with isNegativeℚ factor`. Using `_` for `factor` blocks reduction. Fix: provide explicit argument.
- **Bridge lemma pattern**: Match on same `Dec` as the check function. `yes` branch: `refl ∷ []` or `()`. `no` branch: dual.
- **Composition strategy**: Prove `All E (checkE args)` for each error check, then use `errorIssues-allE-nil` to go from `errorIssues (checkE msgs) ≡ []` to `checkE msgs ≡ []`.
