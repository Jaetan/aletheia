### Root Cause Analysis

The Rosu expansion `Always φ = prog(φ,e) ∧ G(φ)` via `combineAnd` creates a growing `And` tree when the subformula returns `Continue` (i.e., signal is Unknown/not in cache):

```
Frame 0: Always(Atomic p)
Frame 1: And(Atomic p)(Always(Atomic p))          -- 1 And node
Frame 2: And(Atomic p)(And(Atomic p)(Always(Atomic p)))  -- 2 And nodes
Frame n: n And nodes → O(n) per step → O(n²) total
```

This happens when a property monitors a signal on a CAN ID different from the frames being sent (signal never enters the cache, always Unknown).

The old `stepEval` (deleted in 18a) used a fixed-size state machine that didn't grow. The new `stepL` builds a tree via `combineAnd`/`combineOr`.

### Fix Strategy: Rosu Simplification Rules

**Rejected approach**: Change `stepL` to return `Continue(Always φ')` instead of `combineAnd(Continue φ', Continue(Always φ))`.
- This breaks Rosu semantics for non-Atomic subformulas
- For `Always(Until ψ χ)`, `Always(continuation)` is too strong — can produce `Sound True False` (unprovable in adequacy)
- Only correct for Atomic (where φ' = φ)

**Correct approach**: Add Rosu simplification rules (as in Rosu's original paper — rewriting + simplification):
- `And φ (Always φ) → Always φ` when φ structurally equals the Always body
- `Or φ (Eventually φ) → Eventually φ` when φ structurally equals the Eventually body
- Preserves `stepL` and adequacy proof completely
- Applied in `classifyResult` in StreamState (outside proof boundary)
- For Atomic: only needs `_≡ᵇ_` on ℕ (atom index comparison)
- Semantically valid: `⟦ And φ (Always φ) ⟧ ≡ ⟦ Always φ ⟧` (by idempotence of `∧TV`)

**Implementation plan** (not yet started):
1. Add `simplify : LTLProc → LTLProc` to `Coalgebra.agda` (pattern match on specific tree growth patterns)
2. Import `_≡ᵇ_` from `Data.Nat` for atom index comparison
3. Apply `simplify` to `newProc` in `classifyResult` in `StreamState.agda`
4. Type-check both modules, rebuild, re-run benchmarks
5. Also consider full `_≡ᵇ-proc_` (Boolean equality on LTLProc) for handling non-Atomic cases

**Key insight**: The simplification only needs to handle patterns produced by `combineAnd`/`combineOr`. For production use (Always + single Atomic), pattern matching on `And (Atomic n) (Always (Atomic m))` with `n ≡ᵇ m` suffices. Multi-atom formulas like `Always(And (Atomic 0) (Atomic 1))` would need full structural equality on LTLProc.

## Gap D: Semantic Grounding — Adequacy Proof COMPLETE

### What Gap D Is
Neither `stepEval` (incremental evaluator) nor `stepL` (coalgebra) is proven correct against a mathematical definition of LTL. The bisimilarity proof shows they agree with each other, but not that they implement the *right thing*.

### Proof Architecture

Direct adequacy proof (no bisimilarity intermediate layer):
```
stepL ←adequacy→ denotational LTLf
```

### Semantic Contract

The monitor follows **standard RV-LTL semantics** (Bauer-Leucker-Schallhart 2011):
- Safety operators (Always, Release) **propagate Inconclusive** (no premature violation)
- Liveness operators (Eventually, Until) return Inconclusive on Unknown
- Pending atoms trigger retry (Continue) rather than immediate verdict
- **No false positives or false negatives** — monitor only gives definite answers when correct

The relationship between `runL` and `⟦_⟧` is **bilateral Sound** (no wrong definite answers in either direction):
- `Sound True True` — definite agreement
- `Sound False False` — definite agreement
- `Sound m Unknown`, `Sound m Pending` — denotation uncertain, any monitor result OK
- `Sound Unknown d`, `Sound Pending d` — monitor admits ignorance
- NOT `Sound True False`, NOT `Sound False True` — never wrong definite answers

### Proof Architecture Summary

Four-layer proof structure (1061 lines, Adequacy.agda):

1. **Sound compositionality** (`sound-and`, `sound-or`, `sound-not`): compose IH through Kleene connectives
2. **Operational decomposition** (`runL-*-decomp`): `runL table (Op φ ψ) σ ≡ runL table φ σ ⊕TV runL table ψ σ`
3. **Soundness transport** (`sound-transport-monitor`, `sound-transport-denot`): `subst`-based rewriting without `with`-auxiliaries
4. **Non-recursive metric helpers** (`adequacy-met-*`): extract `with` case analysis to fix termination checker

### Key Lessons Learned

- **Nested `with` on two `stepL` results** breaks `combineOr`/`combineAnd` reduction (And-vs-Or issue)
- **Simultaneous `with stepL ψ y | stepL φ y`** works — both values concrete after match, no abstract intermediates
- **`rewrite` in recursive functions** generates `with`-auxiliaries that hide recursive calls from termination checker
- **Non-recursive helpers** with pre-computed IH arguments solve the termination issue
- **`subst`/`sound-transport-*`** avoids `with`-auxiliaries entirely (function application, not pattern matching)
- **Decomposition lemmas inside `with boolean | true`** don't work (normal form mismatch from with-abstraction)
